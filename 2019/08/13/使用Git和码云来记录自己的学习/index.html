使用Git和码云来记录自己的学习

发表于 2019-08-13 更新于 2019-08-28 分类于 Git 评论数： 阅读次数：
本文字数： 1.9k 阅读时长 ≈ 4 分钟

前言

在我们学习了 git 之后，大多数时间我们都单纯用来管理我们的项目，这样并不能让 git 工具得到高效的利用。

养成对学习内容的记录是一个优秀的好习惯，今天我们来使用 git 和码云结合做一个记录学习内容的仓库。

一 、新建码云仓库并编写README文件

1. 新建仓库

首先打开码云官网，登录自己的账号， 点击右上角的新建仓库

仓库名称最好是与你当前学习内容相关的名字，比如在学 JAVA ，这里直接就写 JavaStudyRecord ，下图中红框内容一定要记得勾选：

很明显，我们可以利用这个文件来介绍我们的仓库。那么我们开始修改其中的内容。

码云上的操作基本上结束了，接下来我们在本地上操作。

注意git是不能提交空文件夹的，所以要记得在文件夹里面添加文件。

有过 git 基础的应该都记得，我们要把文件从本地推送到远程，一般需要三个步骤：

  1. 先添加文件(add)，此步骤对应git命令中git add
  2. 提交文件(commit)，此步骤对应git命令中git commit
  3. 推送文件(push)，此步骤对应git命令中git push

注意事项

  1. 新建目录时名字一定要规范

  2. 空文件夹不会被 add 到暂存区，要记得在文件夹里面存放文件才会被提交。

  3. 在每次需要推送文件到远程时，注意要在命令行中进入你本地仓库的根目录，然后执行三件套：

    每次推送完记得检查远程仓库内容是否更新。

  4. 记得每次在本地仓库修改了文件内容都要推送一次，这样既可以在主页看到你的贡献度，有可以看出你自己对学习的态度。
